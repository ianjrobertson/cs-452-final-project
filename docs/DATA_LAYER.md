# Data Layer

## Overview

The data layer is built on Ecto with Postgres. It has three layers:

```
Migrations    → defines the database schema (DDL)
Schemas       → Elixir structs that map to tables + changeset validation
Contexts      → public API functions that all other code calls
```

Nothing outside the context modules touches `Repo` or writes queries directly. Agents, LiveView, and other processes call context functions only.

---

## Migrations

Six migrations, run in order:

| Migration | Table | Notes |
|---|---|---|
| `create_users_auth_tables` | `users`, `users_tokens` | Generated by `phx.gen.auth` |
| `create_markets` | `markets` | Enables pgvector extension |
| `create_signals` | `signals` | Append-only (`updated_at` omitted) |
| `create_briefs` | `briefs` | Append-only (`updated_at` omitted) |
| `create_probability_history` | `probability_history` | No timestamps at all — just `recorded_at` |
| `create_user_subscriptions` | `user_subscriptions` | Composite unique index on `(user_id, market_id)` |

`markets` uses `up/down` instead of `change` because the raw `execute` SQL for pgvector can't be auto-reversed by Ecto.

All FK relationships use `on_delete: :delete_all` — deleting a market cascades to signals, briefs, history, and subscriptions at the DB level.

The IVFFlat index on `signals.embedding` is intentionally deferred — it requires ~1000+ rows to be useful and will be added in a separate migration once real data exists.

## Schemas

One module per table. Each schema defines the struct shape, field types, associations, and changeset validation.

### `Oracle.Markets.Market` (`markets`)
Central entity. Everything else has a `market_id` FK pointing here.
- `condition_id` — Polymarket's unique identifier, used for upserts
- `question_embedding` — 1536-dim vector (pgvector), pre-computed at subscription time, used to score incoming signals
- `has_many` — signals, briefs, probability_history, user_subscriptions

### `Oracle.Markets.UserSubscription` (`user_subscriptions`)
Join table between users and markets.
- `alert_threshold` — float (0.0–1.0), notifies user when probability moves by this amount
- Composite unique constraint on `(user_id, market_id)` — one subscription per user per market

### `Oracle.Signals.Signal` (`signals`)
Every ingested OSINT signal. Append-only.
- `source` — `Ecto.Enum` with values `[:news, :reddit, :economic, :x]`. Stored as string in DB, used as atom in Elixir.
- `embedding` — `Pgvector.Ecto.Vector`, set programmatically after OpenAI call, not in `cast`
- `relevance_score` — cosine similarity against market's `question_embedding`, threshold 0.40 to be stored

### `Oracle.Briefs.Brief` (`briefs`)
LLM-generated intelligence briefs. Append-only. Belongs to a market, not a user.
- `key_signals` — JSONB (`:map` type), snapshot of signals used at generation time
- `probability_at_generation` — snapshot of market probability when brief was made

### `Oracle.Markets.ProbabilityHistory` (`probability_history`)
Time-series price data written by PolymarketAgent on each sync.
- No `timestamps()` — only `recorded_at` (a plain field, not auto-managed)
- No `updated_at` in migration, so none in schema either

## Contexts

### `Oracle.Markets`
Called by: PolymarketAgent, LiveView dashboard

| Function | Description |
|---|---|
| `upsert/1` | Insert or update a market by `condition_id`. Replaces `probability`, `active`, `updated_at` on conflict. |
| `get_by_condition_id/1` | Fetch a market by Polymarket ID. Returns `nil` if not found. |
| `set_question_embedding/2` | Update a market's embedding after OpenAI call. Uses `Ecto.Changeset.change` (skips validation — internal operation). |
| `list_active/0` | All active markets, newest first. Used by browse page. |
| `subscribe/2` | Create a subscription row in a transaction (`Ecto.Multi`). Structured as a transaction so agent room startup can be added as a second step later. |
| `unsubscribe/2` | Delete subscription row. Returns `{:error, :not_found}` if not subscribed. |
| `list_markets_for_user/1` | Inner join markets → subscriptions filtered by user. Returns only subscribed markets. |
| `record_probability/2` | Bulk insert a probability history row via `Repo.insert_all` (bypasses changeset for speed). |
| `probability_history_for/1` | All probability rows for a market, ascending by time. Used by LiveView chart. |

### `Oracle.Signals`
Called by: polling agents (NewsAgent, RedditAgent, EconomicAgent, XAgent)

| Function | Description |
|---|---|
| `insert/1` | Validate and insert a signal. Returns `{:ok, signal}` or `{:error, changeset}`. |
| `top_for_market/2` | Signals for a market with score > 0.6, newest first. Used by SynthesisAgent to build prompt. Default limit 30. |
| `count_since/2` | Count signals for a market since a given datetime. Used by SynthesisAgent threshold gate. |

### `Oracle.Briefs`
Called by: SynthesisAgent, LiveView dashboard

| Function | Description |
|---|---|
| `insert/1` | Validate and insert a brief. |
| `latest_for_market/1` | Most recent brief for a market. Returns `nil` if none exist. Used by SynthesisAgent to check cooldown. |
| `list_for_market/1` | All briefs for a market, newest first. Used by LiveView feed. |

## pgvector Setup

pgvector requires two steps beyond adding the hex dependency:

1. `lib/oracle/postgrex_types.ex` — registers pgvector's type extensions alongside Ecto's Postgres extensions so Postgrex can encode/decode the `vector` binary format
2. `config/config.exs` — `config :oracle, Oracle.Repo, types: Oracle.PostgrexTypes`

Vector fields use `Pgvector.Ecto.Vector` as the field type in schemas. Cosine similarity queries use the `<=>` operator via `fragment` in the query DSL.

## Key Decisions

**Contexts group by domain, not by schema** — `Oracle.Markets` handles markets, subscriptions, and probability history. Subscriptions don't warrant their own context since they have no independent logic.

**`embedding` and `question_embedding` are not in `cast`** — these fields are always set programmatically by agent code, never from user input. Set via `put_change/3` or `Ecto.Changeset.change/2`.

**`Repo.insert_all` for probability history** — PolymarketAgent writes a new row every 2 minutes per market. Bypassing changesets avoids overhead for a high-frequency, internal-only write.

**`Ecto.Multi` for subscribe** — wrapping subscription creation in a transaction now means adding agent room startup as a second step later requires no structural change.

**No IVFFlat index at creation time** — added later in a separate migration once sufficient data exists. Creating it on an empty table provides no benefit and misleads query planner.
